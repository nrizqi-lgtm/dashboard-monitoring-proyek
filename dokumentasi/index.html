Bab 1

Ringkasan Aplikasi

Aplikasi ini adalah Dashboard Monitoring Proyek berbasis web yang dirancang untuk memantau dan mengelola proyek implementasi Smart Campus. Aplikasi menyajikan visualisasi data proyek secara real-time yang mencakup task tracking, KPI akademik, Earned Value Management (EVM), manajemen risiko, anggaran, dan milestone - seluruhnya terhubung ke Google Spreadsheet sebagai basis data melalui Google Apps Script (GAS) yang berperan sebagai backend API.

Dashboard ini dibangun tanpa framework frontend maupun backend konvensional. Seluruh logika tampilan berjalan di sisi klien (browser) menggunakan HTML, CSS, dan JavaScript murni, sementara operasi baca-tulis data didelegasikan ke Google Apps Script yang berinteraksi langsung dengan Google Spreadsheet melalui API internal SpreadsheetApp.

Arsitektur Sistem

Arsitektur aplikasi mengikuti pola three-tier yang disederhanakan: frontend statis di browser, backend berupa Google Apps Script yang di-deploy sebagai Web App, serta Google Spreadsheet sebagai lapisan penyimpanan data. Berikut adalah diagram arsitektur secara keseluruhan:

Gambar 2.1 Diagram Arsitektur Tiga Lapis Aplikasi

Komunikasi antara browser dan Google Apps Script menggunakan dua mekanisme: JSONP untuk operasi pembacaan data (GET) guna mengatasi keterbatasan CORS pada Web App GAS, serta POST dengan Content-Type: text/plain untuk operasi penulisan data (create, update, delete). Tidak ada koneksi langsung dari frontend ke Google Spreadsheet; seluruh operasi data melewati layer GAS.

Bab 4

Alur Data dan Komunikasi

Flow Aplikasi

Berikut adalah alur eksekusi aplikasi dari saat halaman dimuat hingga seluruh modul dirender:

index.html dimuat di browser - menampilkan landing page.
Pengguna menekan tombol "Buka Dashboard" → navigasi ke dashboard.html.
DashboardController.init() dijalankan:
setupEventListeners() - mendaftarkan handler untuk navigasi tab, tombol CRUD, refresh.
google.charts.load() - memuat library Google Charts.
loadAllModules() - menjalankan Promise.all() untuk memuat keenam modul secara paralel.
Setiap modul memanggil SheetsAPI.readSheet(sheetName) → GET request ke GAS.
GAS membaca Spreadsheet → mengembalikan array JSON.
Data modul terisi → Module.render() merender tabel dan chart.
renderOverallProgressGauge() - merender gauge ring di tab Overview.
Auto-refresh diaktifkan setiap 60 detik.

Gambar 4.1 Flowchart Alur Eksekusi Aplikasi

Operasi CRUD

Seluruh operasi data dilakukan melalui endpoint GAS yang dipanggil dari frontend:

Operasi Metode Endpoint / Body Deskripsi
READ GET (JSONP) ?action=getData&sheet=NamaSheet Mengambil seluruh data dari sheet yang ditentukan
CREATE POST {action:'addRow', sheet, data} Menambahkan baris baru ke sheet
UPDATE POST {action:'updateRow', sheet, rowId, data} Memperbarui baris berdasarkan kolom ID
DELETE POST {action:'deleteRow', sheet, rowId} Menghapus baris berdasarkan kolom ID

Tabel 4.1 - Ringkasan Operasi CRUD

Penanganan Kesalahan

Sistem memiliki mekanisme fallback untuk menangani kegagalan komunikasi dengan backend:

  • Jika fetch gagal → sistem menggunakan mock data lokal melalui fungsi getMockData().
  • Jika write (POST) gagal → data disimpan sementara ke localStorage browser.
  • Status koneksi ditampilkan pada badge di top-bar: Connected atau Offline Mode.
Perhatian
Data yang tersimpan di localStorage tidak otomatis disinkronkan kembali ke Spreadsheet saat koneksi pulih. Pengguna perlu secara manual mengulangi operasi write.

Bab 6

Deskripsi Halaman dan Modul

Aplikasi terdiri dari dua halaman utama: Landing Page dan Dashboard. Dashboard utama dibagi menjadi 7 tab yang masing-masing menangani satu aspek monitoring proyek.

Halaman: Landing Page (index.html)

Landing page berfungsi sebagai halaman selamat datang dan portal masuk ke dashboard. Komponen yang ditampilkan meliputi:

  • Logo aplikasi (ikon buku)
  • Judul: "Smart Campus Monitoring"
  • Subtitle yang menjelaskan sistem monitoring terintegrasi berbasis web
  • 4 kartu fitur (masing-masing berisi ikon, judul, dan deskripsi): EVM Analysis, Risk Management, Budget Tracking, Milestones
  • Tombol "Buka Dashboard" yang menavigasi ke dashboard.html

Halaman ini sepenuhnya statis dan tidak memerlukan data dari backend.
